### Block Diagram — DALG / PODEM Module

(See attached SVG / draw.io diagram)

**Input:**
- Levelized circuit netlist (from READ + LEV)
- Single fault specification: (node_num, stuck-at value)
- Heuristic options: D-Frontier selection (-df nl|nh|lh|cc), J-Frontier selection (-jf v0)

**Output:**
- Test pattern file: ternary PI assignment (0 / 1 / x), one vector per fault
- Empty file if fault is redundant or exceeds backtrack limit

**Internal Flow:**
1. Build topology cache (levelize) + compute SCOAP values
2. Inject fault → set fault site to D (SA0) or D' (SA1)
3. DALG path: 5-valued forward implication → D-Frontier propagation → J-Frontier justification (with backtracking)
4. PODEM path: Get objective → Backtrace to PI → Assign + dual-rail simulation → Check PO difference (with backtracking)
5. Ternary compression: drop unnecessary PI assignments using SCOAP ordering

---

### Correctness Verification

1. **Fault simulation cross-check (PFS/DFS):**
   Every test vector generated by DALG/PODEM is fed back into PFS. We verify that the target fault appears in the detected fault list. Function `dalg_pattern_detects_fault()` automates this: it runs good/faulty simulation and checks whether any PO differs.

2. **Golden result comparison (small circuits):**
   For c17 and mini-circuits, we compare fault coverage against TA-provided golden results. Both DALG and PODEM achieve the expected FC.

3. **DALG ↔ PODEM cross-validation:**
   For the same fault, both algorithms may produce different test vectors (due to different search strategies), but both must detect the fault. If one detects and the other doesn't, it indicates a bug. We run both on the full fault list and verify matching FC.

4. **Logic simulation sanity check:**
   The generated test vector is applied via `logicsim` to confirm PO output values match expected behavior under fault-free conditions.

# Slide 6 — ATPG (DALG, PODEM, single fault)
## "What about large circuits?"

---

### Why no golden results for large circuits?

- ATPG is **non-deterministic** — different heuristics, backtrack limits, and search orders produce different (but equally valid) test sets.
- For large ISCAS-85 circuits (c432–c7552), the solution space is vast, so TA cannot provide a single golden answer.

**Key verification steps for large circuits:**
1. Run DALG on full RFL → collect test vectors → run PFS on those vectors → confirm FC matches
2. Repeat with PODEM → confirm FC matches
3. Compare DALG FC vs PODEM FC — they should be very close (small differences due to backtrack limits are acceptable)
4. Check that no "detected" fault is actually undetected by re-simulating each vector independently
